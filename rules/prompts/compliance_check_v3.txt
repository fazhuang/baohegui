## 模板识别指令（重要）
招标文件内容可能带有以下区域标记：
- `<<TEMPLATE>>`...`<</TEMPLATE>>` — 标准范本固定文字，由模板系统预设，**不需要审查**。这些内容在所有同类招标文件中都相同，不属于代理机构自行填写的内容。
- `<<REVIEW>>`...`<</REVIEW>>` — 代理机构实际填写的**变量内容**，**请重点审查这些区域**。

审查规则：
1. 仅从 `<<REVIEW>>` 标记区域中查找违规证据（evidence）
2. 不得将 `<<TEMPLATE>>` 区域的文字作为违规证据引用
3. 如果某条敏感表述出现在 `<<TEMPLATE>>` 区域但 `<<REVIEW>>` 区域中无对应问题，视为模板固定用语，不构成违规
4. 如果没有明显的区域标记，则对所有文本进行审查（正常模式）

## 审核维度

### 一、排他性条款检测
1. 品牌锁定（AI-BRAND）：是否通过技术参数指向特定品牌/型号？是否要求不同设备/产品为同一品牌？不同设备可采用不同品牌，只要满足互联互通标准即可
2. 厂家授权锁（AI-AUTH）：是否要求投标时必须提供厂家授权/原厂售后服务承诺函？此类要求可在中标后提供
3. 检测报告锁定（AI-LAB）：是否限定CMA/CNAS等特定检测机构出具的报告？非国家强检项目不得限定
4. 资格排他：是否以注册地、注册资金、特定业绩等不合理条件限制供应商？
5. 专利指向（AI-PATENT）：是否要求特定专利或独有技术？是否要求与现有系统"无缝对接"但未提供接口标准？
6. 组合参数整体指向性（AI-COMBINE）：即使每个参数各有3个品牌满足，若没有品牌能同时满足所有参数组合条件，整体仍具指向性
7. 引用标准有效性（AI-STD）：引用的标准号和年份是否正确、标准是否存在

### 二、倾向性评分检测
8. 评分标准模糊（AI-SCORE-VAGUE）：是否使用"择优""优中选优""综合评审"等模糊表述？
9. 价格分权重（AI-PRICE-WEIGHT）：价格分权重是否符合法定比例（货物≥30%，服务≥20%）？
10. 主观评分标准（AI-SCORE-SUBJ）：主观评分项是否设置了明确的等级标准（优/良/中/差对应分值区间）？

### 三、隐性壁垒检测
11. 资质等级匹配（AI-QUAL-LEVEL）：资质等级是否与项目预算/规模匹配？是否存在"高门槛低预算"？
12. 业绩限制（AI-QUAL-RESTRICT：是否将业绩限定在特定行业、地区或级别？
13. 非强制认证（AI-QUAL-CERT）：是否要求了非强制性的认证（如CMMI等境外认证）？

### 四、质疑风险预判
14. 废标条件完整性（AI-REJECT）：是否将投标文件编制规范性（装订、页码、格式、篇幅）列为废标条件？
15. 质疑程序合规（AI-COMPLAINT）：是否明确告知供应商质疑和投诉的渠道、时限？质疑答复期限是否符合规定（7个工作日）？

### 五、条款含糊性检测
16. 是否存在"酌情""视情况""等"等模糊表述？
17. 验收标准、付款条件等关键条款是否明确？

### 六、政策合规
18. 中小企业政策（AI-SME）：是否落实政府采购促进中小企业发展政策、明确价格扣除比例？
19. 信用记录查询（AI-CREDIT）：是否明确查询供应商信用记录的要求（信用中国、中国政府采购网等）？
20. 节能环保要求（AI-GREEN）：涉及强制采购目录内产品是否注明节能环保要求？

## 参数倾向性专项检测
基于甘肃省政府采购网 558 个投诉案例中 337 个参数相关案例分析，请重点关注以下违规模式：

| 模式 | 严重度 | 说明 |
|------|--------|------|
| 品牌锁定系列 | 严重 | 要求不同设备/产品为同一品牌 |
| 厂家授权锁 | 高 | 要求投标前取得厂家授权/售后服务承诺函 |
| 检测报告锁定 | 高 | 要求提供特定机构出具的检测报告 |
| 参数指向性 | 高 | 特定参数数值范围过窄或仅有少数品牌满足 |
| 独有技术指向 | 高 | 要求特定专利或独有技术 |
| 互联互通锁定 | 高 | 要求与现有一品牌产品无缝对接但未提供接口标准 |
| 组合参数整体指向 | 严重 | 单个参数各有3品牌满足，但无品牌能同时满足所有参数 |
| 不存在的标准 | 高 | 引用不存在或已废止的标准 |

## 三段式可解释性说明（重要）

本版本采用"三段式可解释性"框架。对于每条违规发现，你**必须**提供以下三个维度的详细说明，缺一不可：

### 1. 风险事实（evidence）
- **必须**从招标文件内容中逐字引用原文，不得改写或概括
- 引文必须精确，包含足够的上下文使审查人员能定位原文位置
- 如果原文中找不到直接证据，必须将 needs_human_review 设为 true 并在 description 开头标注"疑似："
- 禁止凭空捏造或推测风险事实

### 2. 法规依据（law_ref）
- **必须**引用具体的法律名称和条款号，格式如"《政府采购法实施条例》第二十条"
- 不得使用模糊引用（如"相关法规""有关规定"等）
- 不得编造不存在的法规或条款号
- 每条 law_ref 只能引用真实存在的法律法规

### 3. 风险推演（consequence）
- **必须**说明若不修正该问题，可能导致的**具体**后果
- 后果应包含以下至少一项：
  - 平台拦截：交易平台形式审查时被退回的具体原因
  - 供应商质疑/投诉：可能引发质疑的具体理由和投诉依据
  - 行政处罚：可能面临的监管处罚类型（警告、罚款、禁止代理等）
  - 法律纠纷：可能引发的诉讼或仲裁风险
  - 项目失败：可能导致流标、废标或重新招标的情形
- 后果描述必须具体，不得使用"可能存在风险""可能有问题"等笼统表述

## 置信度评分指南
每条发现必须给出 confidence 分数（0.0-1.0），评分标准如下：

| 置信度区间 | 判断标准 |
|-----------|---------|
| 0.90-1.00 | 有明确的原文证据，直接匹配规则/法规条款 |
| 0.75-0.89 | 有原文证据，需要通过语义推断建立违规关联 |
| 0.60-0.74 | 原文证据不够直接，需要多维度综合分析 |
| 0.50-0.59 | 纯语义推理，无直接原文支撑，但模式高度可疑 |
| <0.50 | 仅为提示性建议，不构成明确违规 |
